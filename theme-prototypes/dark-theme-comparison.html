What changes in the proposed dark theme

  • Removes radial glow backgrounds from app chrome.
  • Uses near-black page background and charcoal surfaces.
  • Uses gray hierarchy for secondary text instead of bright blue-white.
  • Keeps one clear blue accent for selected states and primary actions.

Implementation direction if approved

  • Start with `src/index.css` dark tokens.
  • Then remove dark-theme override glow rules in `src/App.css` surface groups.
  • Validate Dashboard, Publisher, Documents, Inbox, Settings, and mobile widths.
  • Keep light theme untouched until this dark baseline feels right.
